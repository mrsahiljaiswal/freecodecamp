Introduction

JavaScript is a versatile programming language primarily used for web development. It allows you to create dynamic and interactive web pages. JavaScript is a high-level, interpreted scripting language that conforms to the ECMAScript specification. It is a multi-paradigm language, supporting event-driven, functional, and imperative programming styles. JavaScript is one of the core technologies of the World Wide Web, alongside HTML and CSS. As of 2021, 97.6% of all websites use JavaScript on the client side for web page behavior, often incorporating third-party libraries.

console.log('Hello, world!');
Syntax

JavaScript syntax is the set of rules that define a correctly structured JavaScript program. JavaScript programs are made up of statements with appropriate syntax. Each statement is an instruction to be executed by the browser. JavaScript statements are composed of values, operators, expressions, keywords, and comments. JavaScript is case-sensitive and uses the Unicode character set.

let x = 10;
let y = 20;
let sum = x + y;
console.log(sum);

JavaScript uses semicolons to separate statements. However, semicolons are optional, and JavaScript will automatically insert them where it thinks they are needed. It is a good practice to use semicolons to avoid potential issues. JavaScript also supports comments, which are ignored by the browser. Comments can be single-line or multi-line.

// Single-line comment
/* Multi-line
   comment */
Variables

Variables are containers for storing data values. In JavaScript, you can declare variables using var, let, or const. The var keyword is used to declare a variable that can be re-assigned. The let keyword is used to declare a block-scoped variable that can also be re-assigned. The const keyword is used to declare a block-scoped variable that cannot be re-assigned.

let name = 'John';
const age = 30;
var isStudent = true;

Variables can store different types of data, such as numbers, strings, booleans, objects, and arrays. JavaScript is a dynamically typed language, meaning that variables can hold values of any data type without specifying the type. Variables can be declared without a value, in which case they will have the value undefined.

let x;
console.log(x); 
x = 10;
console.log(x);
Functions

Functions are blocks of code designed to perform a particular task. They are executed when "called" (invoked). Functions can take parameters and return a value. Functions are defined using the function keyword, followed by the function name, a list of parameters enclosed in parentheses, and the function body enclosed in curly braces.

function greet(name) {
    return 'Hello, ' + name;
}
console.log(greet('Alice'));

Functions can be called multiple times with different arguments. Functions can also be assigned to variables, passed as arguments to other functions, and returned from other functions. JavaScript supports anonymous functions, which are functions without a name. Anonymous functions are often used as arguments to other functions or as immediately invoked function expressions (IIFE).

let greet = function(name) {
    return 'Hello, ' + name;
};
console.log(greet('Bob')); 

(function() {
    console.log('IIFE executed');
})();
Objects

Objects are collections of properties, and a property is an association between a name (or key) and a value. Objects are used to store related data and functionality. In JavaScript, objects are created using curly braces, with properties defined as key-value pairs. Properties can be accessed using dot notation or bracket notation.

let person = {
    firstName: 'John',
    lastName: 'Doe',
    age: 25,
    fullName: function() {
        return this.firstName + ' ' + this.lastName;
    }
};
console.log(person.fullName());

Objects can contain other objects, arrays, and functions. Functions defined within an object are called methods. The this keyword refers to the current object and is used to access properties and methods within the object. Objects can be created using object literals, the new Object() syntax, or constructor functions.

let obj1 = {}; 
let obj2 = new Object(); 

function Person(firstName, lastName, age) {
    this.firstName = firstName;
    this.lastName = lastName;
    this.age = age;
}
let person1 = new Person('Alice', 'Smith', 30);
console.log(person1.firstName);
Arrays

Arrays are used to store multiple values in a single variable. They are a special type of object in JavaScript. Arrays are created using square brackets, with elements separated by commas. Elements can be accessed using their index, which starts at 0. Arrays have a length property that returns the number of elements in the array.

let fruits = ['Apple', 'Banana', 'Cherry'];
console.log(fruits[0]); 
console.log(fruits.length);

Arrays can contain elements of different data types, including numbers, strings, objects, and other arrays. Arrays have built-in methods for adding, removing, and manipulating elements. Common array methods include push(), pop(), shift(), unshift(), splice(), slice(), concat(), forEach(), map(), filter(), and reduce().

let numbers = [1, 2, 3, 4, 5];
numbers.push(6); 
console.log(numbers); 
numbers.pop(); 
console.log(numbers); 
numbers.shift(); 
console.log(numbers); 
numbers.unshift(1); 
console.log(numbers);
Loops

Loops are used to execute a block of code repeatedly. JavaScript supports different types of loops such as for, while, and do-while. The for loop is used to iterate over a block of code a specified number of times. The while loop is used to execute a block of code as long as a specified condition is true. The do-while loop is similar to the while loop, but it executes the block of code at least once before checking the condition.

for (let i = 0; i < 5; i++) {
    console.log(i);
}

let j = 0;
while (j < 5) {
    console.log(j);
    j++;
}

let k = 0;
do {
    console.log(k);
    k++;
} while (k < 5);

JavaScript also supports the for...in loop, which is used to iterate over the properties of an object, and the for...of loop, which is used to iterate over the values of an iterable object, such as an array or a string.

let person = { firstName: 'John', lastName: 'Doe', age: 25 };
for (let key in person) {
    console.log(key + ': ' + person[key]);
}

let fruits = ['Apple', 'Banana', 'Cherry'];
for (let fruit of fruits) {
    console.log(fruit);
}
Events

Events are actions that occur in the browser, such as clicks, key presses, or mouse movements. JavaScript can be used to handle these events. Event handling is done using event listeners, which are functions that are called when an event occurs. Event listeners can be added to HTML elements using the addEventListener() method.

document.getElementById('myButton').addEventListener('click', function() {
    alert('Button clicked!');
});

Event listeners can be used to handle different types of events, such as click, mouseover, keydown, and submit. Event listeners can also be removed using the removeEventListener() method. Event objects are passed to event listeners and contain information about the event, such as the target element, the type of event, and the coordinates of the mouse pointer.

document.getElementById('myElement').addEventListener('mouseover', function(event) {
    console.log('Mouse over:', event.target);
});
document.addEventListener('keydown', function(event) {
    console.log('Key pressed:', event.key);
});
document.getElementById('myForm').addEventListener('submit', function(event) {
    event.preventDefault(); 
    console.log('Form submitted');
});
Promises

Promises are used to handle asynchronous operations in JavaScript. They represent a value that may be available now, in the future, or never. A promise is an object that represents the eventual completion (or failure) of an asynchronous operation and its resulting value. Promises have three states: pending, fulfilled, and rejected. A promise is created using the Promise constructor, which takes a function with two arguments: resolve and reject.

let promise = new Promise((resolve, reject) => {
    let success = true;
    if (success) {
        resolve('Operation successful');
    } else {
        reject('Operation failed');
    }
});
promise.then(result => {
    console.log(result);
}).catch(error => {
    console.log(error);
});

Promises can be chained using the then() method, which takes two arguments: a callback function for the fulfilled state and a callback function for the rejected state. The catch() method is used to handle errors. The finally() method is used to execute code regardless of the promise's outcome. Promises can be combined using the Promise.all() and Promise.race() methods.

let promise1 = new Promise((resolve, reject) => {
    setTimeout(() => resolve('Promise 1 resolved'), 1000);
});
let promise2 = new Promise((resolve, reject) => {
    setTimeout(() => resolve('Promise 2 resolved'), 2000);
});
Promise.all([promise1, promise2]).then(results => {
    console.log(results); 
}).catch(error => {
    console.log(error);
});
Async Await

Async/Await is a syntactic sugar built on top of promises. It allows you to write asynchronous code in a more synchronous fashion. The async keyword is used to declare an asynchronous function, which returns a promise. The await keyword is used to wait for a promise to be resolved or rejected. The await keyword can only be used inside an async function.

async function fetchData() {
    try {
        let response = await fetch('https://api.example.com/data');
        let data = await response.json();
        console.log(data);
    } catch (error) {
        console.log('Error:', error);
    }
}
fetchData();

Async functions can contain multiple await expressions, which are executed sequentially. The await keyword pauses the execution of the async function until the promise is resolved or rejected. If the promise is resolved, the await expression returns the resolved value. If the promise is rejected, the await expression throws an error, which can be caught using a try...catch block.

async function fetchMultipleData() {
    try {
        let response1 = await fetch('https://api.example.com/data1');
        let data1 = await response1.json();
        console.log(data1);

        let response2 = await fetch('https://api.example.com/data2');
        let data2 = await response2.json();
        console.log(data2);
    } catch (error) {
        console.log('Error:', error);
    }
}
fetchMultipleData();
DOM Manipulation

The Document Object Model (DOM) is a programming interface for HTML and XML documents. It represents the page so that programs can change the document structure, style, and content. The DOM is a tree-like structure, where each node represents an element, attribute, or text. JavaScript can be used to manipulate the DOM by selecting elements, modifying their attributes, and changing their content.

document.getElementById('myElement').textContent = 'Hello, DOM!';

Elements can be selected using methods such as getElementById(), getElementsByClassName(), getElementsByTagName(), querySelector(), and querySelectorAll(). Attributes can be modified using methods such as setAttribute(), getAttribute(), and removeAttribute(). Content can be changed using properties such as textContent, innerHTML, and outerHTML.

let element = document.querySelector('.myClass');
element.setAttribute('data-custom', 'value');
console.log(element.getAttribute('data-custom')); 
element.removeAttribute('data-custom');
element.textContent = 'New content';
element.innerHTML = 'Bold content';
Fetch API

  • The Fetch API provides a modern way to make network requests. It is more powerful and flexible than the older XMLHttpRequest. The Fetch API is based on promises and provides a more powerful and flexible feature set. The fetch() function takes a URL as an argument and returns a promise that resolves to the response object representing the response to the request.

    fetch('https://api.example.com/data')
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.log('Error:', error));

    The response object contains information about the response, such as the status code, headers, and body. The body can be read using methods such as text(), json(), blob(), and arrayBuffer(). The fetch() function can also take an optional second argument, which is an options object that allows you to specify the request method, headers, body, and other settings.

    fetch('https://api.example.com/data', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({ key: 'value' })
})
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.log('Error:', error));
    ES6 Features

    ES6, also known as ECMAScript 2015, introduced many new features to JavaScript, such as let and const, arrow functions, template literals, and more. The let and const keywords are used to declare block-scoped variables. Arrow functions provide a shorter syntax for writing function expressions and do not have their own this value. Template literals allow you to create multi-line strings and embed expressions using backticks and the ${} syntax.

    let name = 'John';
const age = 30;
let greeting = Hello, my name is ${name} and I am ${age} years old.;
console.log(greeting);

    ES6 also introduced other features such as default parameters, rest and spread operators, destructuring assignment, classes, modules, and more. Default parameters allow you to specify default values for function parameters. The rest operator allows you to represent an indefinite number of arguments as an array. The spread operator allows you to expand an array or object into individual elements. Destructuring assignment allows you to extract values from arrays or objects into variables.

    function greet(name = 'Guest') {
    return Hello, ${name};
}
console.log(greet()); 
console.log(greet('Alice')); 

function sum(...numbers) {
    return numbers.reduce((total, num) => total + num, 0);
}
console.log(sum(1, 2, 3)); 

let arr1 = [1, 2, 3];
let arr2 = [...arr1, 4, 5, 6];
console.log(arr2); 

let person = { firstName: 'John', lastName: 'Doe', age: 25 };
let { firstName, lastName } = person;
console.log(firstName); 
console.log(lastName);
